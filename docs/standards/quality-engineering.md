## RAVL 8-Pillar Methodology
1. Living Acceptance Criteria (AC IDs)
2. Permutation Matrices (Edge cases)
3. Performance Audits
4. SOLID Checks
5. Unit/Integration Tests
6. Regression Tests
7. Stress Tests
8. Security Audits

## Pillar 1: Living ACs
Every change must have corresponding ACs indocs/ACs/ before implementation.
Use Given/When/Then format with unique IDs (AC-PREFIX-NUMBER).
